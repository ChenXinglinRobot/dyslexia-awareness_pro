### Commit 建议

```
fix(modal): Stack drag cards position drift in FamousDyslexicsModal

Root cause: two independent issues caused cards to render at wrong
viewport positions outside the Sheet.

Fix 1 — delayed Stack mount (FamousDyslexicsModal.tsx):
Framer Motion's drag projection measurement ran during the Sheet's
CSS slide-in animation (duration-500), recording incorrect bounding
boxes. Wait 550ms (500ms animation + 50ms buffer) after Sheet opens
before mounting <Stack>, so projection measures against the final
layout. Uses the same useState/setTimeout pattern as ActionSection.

Fix 2 — translate compensation (Stack.css):
Added .card-rotate .card { translate: -50% -50% } to counter the
visual drift from transformOrigin: 90% 90% + scale. Uses the modern
independent translate property — not overridden by Framer Motion's
inline transform shorthand. Scoped to .card-rotate .card to avoid
leaking into CardSwap's .card elements.

Also removed diagnostic !important CSS overrides in
FamousDyslexicsModal.css (position/height/overflow/perspective).

Impact: Stack.tsx unchanged. Only FamousDyslexicsModal (sole Stack
consumer) and Stack.css affected. CardSwap verified unaffected via
scoped selector. Build passes cleanly.
```

### 根因确认

**根因 1**（卡片飞到 Sheet 外）：Framer Motion `drag` 指令在 mount 时执行 projection 测量。Stack 在 Sheet `open` 后立即渲染，此时 Sheet 的 CSS `slide-in-from-left` 动画（duration-500）尚未结束，`motion.div` 获取了错误的视口边界框，注入巨大 `translate` 偏移试图"补偿"位置差。

**根因 2**（卡片视觉偏移）：`motion.div` 的 `animate={{ rotateZ, scale, transformOrigin: '90% 90%' }}` 让卡片从右下角缩放/旋转，视觉重心偏移。传统 CSS 居中方案 `transform: translate(-50%, -50%)` 被内联 `transform` 完全覆盖。改用独立 `translate` 属性解决。
